# ARCHITECTURE — `habesha-names`

**Ethiopian/Eritrean name intelligence for Python: parsing, transliteration, variant generation, and patronymic-aware fuzzy matching.**

Version: 0.1 draft · Owner: Robel · License: MIT · PyPI target: `habesha-names` (import `habesha_names`)

---

## 1. Problem

Habesha names break every global identity system:

1. **No family names.** Full name = given name + father's given name (+ grandfather's). "First/Last name" fields are semantically wrong.
2. **No standard romanization.** ጸሐይ → Tsehay / Tsehai / Sehay / Tzehay. Same person, four database records.
3. **Compound given names.** "Haile Mariam" can be ONE given name (Hailemariam) or given + patronym. Naive tokenizers guess wrong.
4. **Abbreviation conventions.** Gebremedhin → G/Medhin, G.Medhin, Gebre Medhin — all common in official documents.
5. **Fidel homophones.** ሀ/ሐ/ኀ, ሰ/ሠ, ጸ/ፀ, አ/ዐ are pronounced identically; spelling varies by writer.

Target users: KYC/AML & sanctions screening, remittance platforms, HR/payroll, airlines, humanitarian registration, dedup/entity-resolution pipelines.

## 2. Design principles (non-negotiable)

| Principle | Why |
|---|---|
| **Zero runtime dependencies** (stdlib only) | Enterprise adoption, no supply-chain review friction |
| **Deterministic** — no ML at runtime, no network calls | Compliance/audit requirements |
| **Explainable matching** — every score ships an explanation object | AML analysts must justify matches |
| **Data as reviewable assets** — all linguistic data in JSON with `verified` flags | AI agent must never ship unreviewed hallucinated linguistic data |
| **Typed** (`py.typed`, mypy strict) | Library consumers |
| **Fast** — tables precompiled at import, pure functions | Batch dedup over millions of records |

Python ≥ 3.9. Dev-only deps: pytest, ruff, mypy.

## 4. Module design

### 4.1 `fidel` — Ethiopic script core

- **Unicode coverage:** Ethiopic U+1200–U+137F, Supplement U+1380–U+139F, Extended U+2D80–U+2DDF.
- **Syllable model:** each fidel = consonant series + vowel order (1=ä, 2=u, 3=i, 4=a, 5=é, 6=ə/bare, 7=o, plus labialized forms). Series are 8-codepoint aligned in most of the block → decomposition is arithmetic: `order = (cp - series_base)`, table maps series_base → consonant.
- **`tables.py` is generated by `scripts/gen_fidel_tables.py`, never hand-typed.** Hand-typed Ethiopic tables are the #1 hallucination risk. Generator derives from codepoint arithmetic + a small hand-verified seed of series bases; output committed to repo with a `# GENERATED — do not edit` header. Spot-check tests pin known glyphs (ሀ=U+1200 h-1st, ቤ=U+1264 b-5th, ጽ=U+133D ts'-6th).
- **`normalize.py`:** collapse homophone series before any comparison:
  - ሀ-series / ሐ-series / ኀ-series → ሀ-series (same vowel order preserved)
  - ሠ → ሰ · ፀ → ጸ · ዐ → አ
  - strip Ethiopic punctuation (።, ፣ …), normalize whitespace, NFC.

### 4.2 `translit` — transliteration + variants

- **Schemes** (per-consonant, per-vowel-order mapping tables in `schemes.py`):
  - `practical` (default): how Ethiopians actually romanize names — no diacritics. This is the workhorse.
  - `bgn_pcgn`, `academic` (Wright/LOC-style with diacritics): for interoperability with library/government datasets. v0.2.
- **`variants.py` — the crown jewel.** Input: name in fidel OR Latin. Output: ranked set of plausible Latin spellings. Rule engine (ordered rewrite rules), including:
  - `ts↔s↔tz` (ጸ), `q↔k` (ቀ), `h↔kh` (ኀ)
  - 6th-order vowel ambiguity: `Gäbrä → Gebre|Gabre|Gebra`
  - terminal `-ay|-ai|-aye` (Tesfay/Tesfai/Tesfaye)
  - optional gemination doubling (Kebede/Kebbede, Alemu/Allemu)
  - `w↔ou`, `é↔ie↔e` (Bethlehem/Betelhem)
  - Arabic-origin name table: Mohammed/Mohamed/Muhammed/Mohammad/Mahamed, Hussein/Husen/Hussen, Fatuma/Fatima
  - compound splits/joins/abbreviations: Gebremedhin ↔ Gebre Medhin ↔ Gebre-Medhin ↔ G/Medhin ↔ G.Medhin
  - Cap on combinatorial explosion: rules carry weights; emit top-N by cumulative likelihood (default N=25).
- **`to_fidel.py` (v0.2):** reverse transliteration, returns ranked candidates (Latin→fidel is one-to-many; never pretend otherwise).

### 4.3 `parse` — name structure

`parse(raw: str, *, assume_diaspora: bool = False) -> ParsedName`

```python
@dataclass(frozen=True)
class ParsedName:
    raw: str
    title: str | None          # Ato, W/ro, Dr, Qes, Sheikh, Haji, ...
    given: str                 # may be compound: "Hailemariam"
    patronym: str | None
    avonym: str | None         # grandfather
    given_is_compound: bool
    compound_confidence: float # 0–1; "Haile Mariam Desalegn" is ambiguous
    script: Literal["ethiopic", "latin", "mixed"]
    has_surname: Literal["no", "unknown"]  # "unknown" if assume_diaspora
    notes: list[str]
```

Logic order: strip/record title → detect script → expand slash-abbreviations via compound lexicon (G/Medhin: `G/` + known second element ⇒ candidates Gebre-, Girma- weighted by corpus frequency) → compound detection (prefix lexicon: Gebre, Wolde, Haile, Tekle, Kidane, Zera, Amde, Fikre, Berhane, Welete(f)… × second elements: Mariam, Michael, Giorgis, Selassie, Medhin, Iyesus, Kristos, Egziabher, Hiwot, Haymanot, Ab, Tsadik…) → positional role assignment (given/patronym/avonym) → handle "Surname, Given" comma inversion and initials ("Abebe B.").

Diaspora mode: patronym may be an inherited legal surname; parser flags rather than decides.

### 4.4 `match` — the money module

- **`phonetic.py` — HabeshaKey:** Soundex-family algorithm tuned for romanized Habesha names. Sketch: lowercase → strip non-alpha → digraph folding (ts/tz→S, sh→X, ch→C, kh/gh→H, ph→F, th→T) → collapse doubles → terminal `-aye/-ai/-ay → A` → encode consonant skeleton + first vowel. Exact rules tuned against the golden corpus (Task 8), not decided a priori.
- **`token.py`:** `sim(a, b) = max(phonetic_exact*w1, jaro_winkler(norm_a, norm_b), variant_set_overlap)`. Jaro-Winkler implemented in-repo (stdlib-only rule).
- **`full.py`:** full-name scoring:
  1. Parse both names; strip titles; expand abbreviations; normalize.
  2. Build token similarity matrix.
  3. Optimal assignment (n≤4 → brute force is fine) with **swap-tolerance** (given/patronym order swaps common across systems) and **truncation-tolerance** (2-token vs 3-token: unmatched avonym = mild penalty, not failure).
  4. Positional weights: given 0.45, patronym 0.35, avonym 0.20 (config dataclass, overridable).
  5. Return:

```python
@dataclass(frozen=True)
class MatchResult:
    score: float               # 0–1
    pairs: list[TokenPair]     # (token_a, token_b, sim, method)
    swapped: bool
    notes: list[str]           # "avonym missing in b", "G/Medhin expanded", ...
    def __float__(self): ...   # so `match(a,b) > 0.9` just works
```

### 4.5 `data` — curated lexicons

`given_names.json` entry contract:

```json
{
  "fidel": "ፀሐይ",
  "canonical": "Tsehay",
  "variants": ["Tsehai", "Sehay", "Tzehay"],
  "gender": {"f": 0.97, "m": 0.03},
  "origin": "amharic",
  "freq_tier": 1,
  "verified": false
}
```

**Rule: the coding agent may seed entries but every entry ships `"verified": false` until Robel (native speaker) flips it.** `variants()` and `match()` work without the lexicon (rule-based); the lexicon boosts precision. v0.1 target: 300 high-frequency names; v0.2: 2,000+.

## 6. Quality gates

- **Golden corpus** `tests/golden/pairs.json`: ≥150 curated pairs — (same-person spelling variants → expect score ≥ 0.85), (different people, similar names e.g. Alemu/Almaz, Tesfaye/Tesfa → expect ≤ 0.6). CI fails on regression.
- **Property tests:** every output of `variants(x)` must `match(x, v) ≥ 0.8`; `normalize` idempotent; `parse(str(parsed))` stable.
- **Benchmarks:** `scripts/benchmark.py` — target ≥ 50k `match()` calls/sec single core.
- CI matrix: Python 3.9–3.13, ruff, mypy --strict, pytest.
